Bug Reports:
Bug Report: Missing Required Field Validation on Application Form

The "Personal Information" section of the application form identifies several fields as mandatory using an asterisk (*). However, the system allows the user to click the "Next: Your Story" button and proceed to the next step without entering any data into these fields or displaying an error message.

Steps to Reproduce
Navigate to the application URL.
Leave the "Pseudonym (Creative Identity)" field blank.
Leave the "Date of Birth" field blank.
Scroll to the bottom and click the "Next: Your Story" button.
Expected Result
The form should prevent navigation. Inline error messages (e.g., "This field is required") should appear in red below the empty mandatory fields, and the focus should jump to the first missing entry.
Actual Result
The system ignores the "required" status and attempts to load the next section of the form despite the missing information.



Bug Report: Forgot Password link isn’t working

As a user I clicked the forgot your  password link and entered my email address then wait some time, but no email message was received.

Bug Report: "Export Compliance PDF" Button Inactive
The "Export Compliance PDF" button on the Operations Dashboard is currently non-functional. Clicking the button results in no action: no PDF is generated, no download is triggered, and no error message is displayed to the user.
Steps to Reproduce
Log in to the The Indie Quill Collective portal as an Admin.
Navigate to the Dashboard in the top navigation bar.
Select the Operations tab within the Admin Dashboard.
Click the green "Export Compliance PDF" button located in the top-right of the Operations section.







Bug Report: Missing Input Validation on Guardian Contact Fields
The Guardian Email and Guardian Phone fields lack proper input validation. Users are currently able to submit or enter invalid data formats (such as non-numeric characters in the phone field and non-email strings in the email field) without triggering an error message or restriction.
Steps to Reproduce
Navigate to the intake form where "Guardian Information" is required (triggered by a Date of Birth indicating a minor).
In the Guardian Email field, enter a string that does not follow standard email formatting (e.g., "rt34554few").
In the Guardian Phone field, enter an excessively long string containing both numbers and letters (e.g., "12323432...nghjhgjgjk").
Observe that the fields do not display validation errors or block the invalid input.



Bug Report: "Advance" Action Does Not Persist After Refresh
When a user clicks the "Advance" action on a manuscript in the Publishing Pipeline, the "Stage" status updates visually in the UI. However, this change is not being persisted to the database. Upon clicking the "Refresh" button or reloading the page, the manuscript reverts to its previous stage.
Steps to Reproduce
Navigate to the Publishing tab and ensure the Pipeline sub-tab is selected.
Identify a manuscript (e.g., "Legacy Work #2" currently in the Editing stage).
Click the "Advance >" link in the Action column.
Observe the Stage label change visually (e.g., from Editing to Vaulted).
Click the "Refresh" button located at the top right of the Publishing Pipeline section.
Observe the manuscript's stage.


Bug Report: Student Role Permissions Not Enforced
Users assigned the Student role are not being restricted to the appropriate student-facing interface. Despite being marked as an "Applicant" in the backend, the user's view includes navigation items and pages intended for advanced or internal use, rather than a restricted intake or student dashboard.
Steps to Reproduce

Log in as that specific student user ("Miia").
Observe the top navigation bar and accessible pages.
